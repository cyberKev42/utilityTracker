Create `backend/services/sectionsService.js` with these exports:

1. `getAll(userId, { includeArchived = false })` — Query utility_sections LEFT JOIN utility_meters WHERE user_id=$1. Filter `s.archived_at IS NULL` and `m.archived_at IS NULL` unless includeArchived. ORDER BY s.sort_order, m.sort_order. Return sections array with nested `meters` array (group in JS, not SQL). Each section: `{ id, name, unit, icon, sort_order, archived_at, created_at, updated_at, meters: [{ id, name, entry_mode, sort_order, archived_at }] }`.

2. `create(userId, { name, unit, icon })` — INSERT into utility_sections with sort_order = `COALESCE((SELECT MAX(sort_order)+1 FROM utility_sections WHERE user_id=$1), 0)`. RETURNING *.

3. `update(userId, sectionId, { name, unit, icon })` — UPDATE utility_sections SET name=$3, unit=$4, icon=$5, updated_at=now() WHERE id=$2 AND user_id=$1 RETURNING *. Return null if no rows affected.

4. `remove(userId, sectionId)` — DELETE FROM utility_sections WHERE id=$1 AND user_id=$2. CASCADE handles meters+entries. Return boolean (rowCount > 0).

5. `archive(userId, sectionId)` — Use transaction (pool.connect + BEGIN/COMMIT/ROLLBACK pattern from RESEARCH.md). UPDATE utility_sections SET archived_at=now() WHERE id=$1 AND user_id=$2. Then UPDATE utility_meters SET archived_at=now() WHERE section_id=$1. Return updated section or null.

6. `unarchive(userId, sectionId)` — Transaction. SET archived_at=NULL on section and all its meters. Return updated section or null.

7. `reorder(userId, orderIds)` — Use unnest batch update pattern: `UPDATE utility_sections SET sort_order=updates.new_order FROM (SELECT unnest($1::uuid[]) AS id, generate_series(0, $2) AS new_order) AS updates WHERE utility_sections.id=updates.id AND utility_sections.user_id=$3`. Parameters: [orderIds, orderIds.length - 1, userId].

8. `createMeter(userId, sectionId, { name })` — First verify section ownership (SELECT id FROM utility_sections WHERE id=$1 AND user_id=$2). Then INSERT into utility_meters with section_id, name, sort_order=COALESCE(MAX+1, 0). RETURNING *.

9. `updateMeter(userId, meterId, { name })` — UPDATE utility_meters m SET name=$3, updated_at=now() FROM utility_sections s WHERE m.id=$2 AND m.section_id=s.id AND s.user_id=$1 RETURNING m.*. Return null if no rows.

10. `removeMeter(userId, meterId)` — DELETE FROM utility_meters m USING utility_sections s WHERE m.id=$1 AND m.section_id=s.id AND s.user_id=$2. CASCADE handles entries. Return boolean.

11. `reorderMeters(userId, sectionId, orderIds)` — Verify section ownership first. Then unnest batch update on utility_meters WHERE section_id=$1.

12. `getLastReading(userId, meterId)` — Ownership check via JOIN. Then SELECT * FROM utility_entries WHERE meter_id=$1 ORDER BY date DESC, created_at DESC LIMIT 1. Return row or null.

All functions start with `const pool = getDb(); if (!pool) throw new Error('Database not configured');`

1. `getAll(req, res)` — Parse `req.query.include_archived === 'true'`. Call `sectionsService.getAll(req.user.id, { includeArchived })`. Return `res.json(sections)` (200). Catch errors: 503 if DB unavailable, 500 otherwise.

2. `create(req, res)` — Validate: `name` required (string, 1-100 chars), `unit` required (string, 1-20 chars), `icon` optional (string, max 50 chars). Return 400 `{ error: 'Required fields: name, unit' }` if missing. Call service.create. Return 201 with section.

3. `update(req, res)` — Validate `req.params.id` matches UUID_REGEX (400 if not). At least one of name/unit/icon must be provided (400 if empty update). Call service.update. Return 404 `{ error: 'Section not found' }` if null. Return 200 with section.

4. `remove(req, res)` — Validate UUID param. Call service.remove. Return 404 if false. Return 204 (no body) if true.

5. `archive(req, res)` — Validate UUID param. Call service.archive. Return 404 if null. Return 200 with section.

6. `unarchive(req, res)` — Same pattern as archive but calls service.unarchive.

7. `reorder(req, res)` — Validate `req.body.order` is array of UUID strings (400 if not array or any element fails UUID_REGEX). Call service.reorder. Return 200 `{ success: true }`.

8. `createMeter(req, res)` — Validate section UUID param. Validate `name` required (1-100 chars). Call service.createMeter. Return 404 if section not found. Return 201 with meter.

9. `updateMeter(req, res)` — Validate meter UUID param. Validate `name` required. Call service.updateMeter. Return 404 if null. Return 200 with meter.

10. `removeMeter(req, res)` — Validate meter UUID param. Call service.removeMeter. Return 404 if false. Return 204.

11. `reorderMeters(req, res)` — Validate section UUID param. Validate `req.body.order` is UUID array. Call service.reorderMeters. Return 200 `{ success: true }`.

12. `getLastReading(req, res)` — Validate meter UUID param. Call service.getLastReading. Return 404 `{ error: 'No entries found for this meter' }` if null. Return 200 with entry.

Error handling pattern for all: `try { ... } catch (error) { if (error.code === 'ECONNREFUSED' || error.message?.includes('not configured')) return res.status(503).json({ error: 'Database unavailable' }); res.status(error.status || 500).json({ error: error.message || 'Internal server error' }); }`

```javascript
import { Router } from 'express';
import * as sectionsController from '../controllers/sectionsController.js';
import { authenticate } from '../middleware/auth.js';

const router = Router();
router.use(authenticate);

// Section CRUD
router.get('/', sectionsController.getAll);
router.post('/', sectionsController.create);
router.put('/reorder', sectionsController.reorder);
router.put('/:id', sectionsController.update);
router.delete('/:id', sectionsController.remove);
router.post('/:id/archive', sectionsController.archive);
router.post('/:id/unarchive', sectionsController.unarchive);

// Meter CRUD (nested under section)
router.post('/:id/meters', sectionsController.createMeter);
router.put('/:id/meters/reorder', sectionsController.reorderMeters);
router.put('/:sectionId/meters/:meterId', sectionsController.updateMeter);
router.delete('/:sectionId/meters/:meterId', sectionsController.removeMeter);

// Meter last reading (convenience endpoint)
router.get('/meters/:meterId/last-reading', sectionsController.getLastReading);

export default router;
```

IMPORTANT: Place `/reorder` BEFORE `/:id` so Express doesn't match "reorder" as an ID. Same for `/meters/reorder` before `/meters/:meterId`.
