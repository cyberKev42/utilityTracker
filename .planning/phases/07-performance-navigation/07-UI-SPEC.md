## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding |
| sm | 8px | Compact element spacing, icon-to-label gaps |
| md | 16px | Default card padding, form field spacing |
| lg | 24px | Section padding, card gap in grid |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Touch targets (input, select, button): minimum 44px height — already enforced in index.css for input[type="date"], input[type="number"], select

## Color

All values are dark-only (color-scheme: dark; no light mode). Sourced from index.css CSS custom properties.

| Role | Value (HSL) | Usage |
|------|-------------|-------|
| Dominant (60%) | hsl(0 0% 3.5%) — `--background` | Page background |
| Secondary (30%) | hsl(0 0% 6.5%) — `--card` | Card surfaces, section cards, dialogs |
| Accent (10%) | hsl(217 91% 60%) — `--primary` (blue) | Specific interactive elements listed below |
| Destructive | hsl(0 72% 51%) — `--destructive` | Delete button, destructive confirmation actions only |

**Accent reserved for:**
- Sparkline line stroke on dashboard section cards (the curve itself)
- Hover ring/outline on the sparkline icon container (indicating clickability — D-13)
- "Back to Statistics" button on the section stats page (primary variant)
- Focus ring on form inputs (`--ring` matches `--primary`)

Accent is NOT used for: general card borders, muted labels, section color badges (those use per-section Tailwind color classes: amber-500, sky-500, orange-500, emerald-500, violet-500, rose-500).

Supporting muted colors used in this phase:
- `hsl(0 0% 45%)` (`--muted-foreground`) — sparkline container border at rest, secondary text
- `hsl(0 0% 12%)` (`--border`) — card and input borders
- `hsl(0 0% 60% / 0.6)` — delete button text at opacity-60 during deleting state

## Interaction States

### Sparkline on Dashboard Cards (D-10, D-11, D-13)

| State | Visual |
|-------|--------|
| Rest | 48×24px Recharts LineChart, no axes, no tooltip, no grid, stroke = `hsl(217 91% 60%)` (primary blue), strokeWidth 1.5, dot=false |
| Hover | Container gets `ring-1 ring-primary/40` (subtle blue outline); cursor: pointer |
| Active/pressed | opacity-80 on container |
| Data loading (first visit) | Render skeleton bar same dimensions (48×24px, `bg-muted rounded-sm animate-pulse`) |
| Empty data (no entries for section) | Flat line at midpoint — Recharts renders gracefully with a single zero point |

Sparkline data key: `usage_amount` (default per RESEARCH.md open question recommendation — unit-based, more concrete than cost at icon scale).

Data shape passed to Recharts: `[{ value: number }, ...]` — normalize from `deriveWeeklyData` output to use key `value` for `dataKey="value"` in `<Line>`.

### Delete Button Animation (D-07, D-08)

| State | Visual |
|-------|--------|
| Rest | Standard destructive Button variant — `bg-destructive text-destructive-foreground` |
| Deleting (awaiting server) | `opacity-60`, `disabled`, Framer Motion pulse on label text: `animate={{ opacity: [1, 0.4, 1] }}`, `transition={{ repeat: Infinity, duration: 1 }}`, label text changes to "Deleting…" |
| Server confirmed | Item removed from list; button returns to rest or unmounts |
| Server error (after retries) | Button returns to rest state; item remains in list; silent failure (no toast needed — D-06 covers retry) |

### Optimistic Create/Update (D-05)

| State | Visual |
|-------|--------|
| Optimistic (< 300ms) | Item appears immediately in list with real-looking data; no spinner, no placeholder |
| Server confirms | State replaced with server response — no visual change if data matches |
| Server error (after retries) | Item disappears (create rollback) or reverts to previous value (update rollback); no error UI required beyond silent rollback |

### First-Visit Loading State (D-02)

| State | Visual |
|-------|--------|
| First Dashboard visit (loading=true) | Render skeleton cards — `bg-card animate-pulse rounded-lg` at the same dimensions as section cards |
| Subsequent visits (loading=false) | Render immediately from cached context; no skeleton, no delay |

Choice: loading skeleton (not blank space) — per Claude's Discretion. Consistent with existing card dimensions. Avoids layout shift.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — section stats back button | "Back to Statistics" (D-12) |
| Sparkline hover tooltip (none) | No tooltip — sparkline is a visual indicator only, not interactive data |
| Delete button — resting | "Delete" |
| Delete button — deleting state | "Deleting…" |
| Empty state — sparkline with no data | No copy — render flat sparkline line (graceful Recharts fallback); no text label at icon scale |
| Error state — mutation rollback | Silent — no visible error copy on optimistic rollback failure per D-06 (silent retry then revert) |
| First-visit loading — section cards | No copy in skeleton state — structural skeleton only (shape, no text placeholders) |

No destructive confirmation dialogs in this phase. Deletes already exist in the app with existing dialogs — this phase only adds a loading animation to the delete button (D-08), not a new confirmation pattern.
