## Spacing Scale

All values are Tailwind utility classes derived from the 8-point scale. The project does not define custom spacing tokens — use Tailwind defaults (1 unit = 4px).

| Token | Value | Tailwind Class | Usage in This Phase |
|-------|-------|----------------|---------------------|
| xs | 4px | gap-1 / p-1 | Icon gaps inside card header, drag handle margin |
| sm | 8px | gap-2 / p-2 | Meter row internal padding, icon picker cell padding |
| md | 16px | gap-4 / p-4 | Card content default horizontal padding |
| lg | 20px | p-5 | Card content padding (matches existing Settings cards: `p-5`) |
| xl | 24px | gap-6 | Space between section cards, archived area top margin |
| 2xl | 32px | mt-8 | Space between card groups on Settings page |
| 3xl | 48px | — | Not used in this phase |

Exceptions:
- Touch targets for interactive controls (drag handle, delete icon, edit icon): minimum 44px height, enforced via `min-h-[44px]` — consistent with existing `index.css` rule for `input[type="date"]` etc.
- Icon picker grid cells: 40px × 40px (`w-10 h-10`) — below touch minimum but acceptable inside a dialog scrollable area.

## Color

Colors are CSS custom properties resolved at runtime from `index.css :root`. All values below are resolved HSL equivalents for reference only — always use Tailwind semantic tokens in code.

| Role | CSS Variable | Resolved (approx) | Tailwind Token | Usage |
|------|-------------|-------------------|----------------|-------|
| Dominant (60%) | --background | hsl(0 0% 3.5%) — near-black | bg-background | Page background, modal overlay backdrop |
| Secondary (30%) | --card | hsl(0 0% 6.5%) — dark grey | bg-card | Section cards, dialog body, archived area container |
| Accent (10%) | --primary | hsl(217 91% 60%) — blue | text-primary / bg-primary | See reserved list below |
| Destructive | --destructive | hsl(0 72% 51%) — red | text-destructive / bg-destructive | Permanent delete button, delete confirmation dialog confirm button |

**Accent (--primary / blue) reserved for:**
- Card group icon background: `bg-primary/10` with `text-primary` icon (matches Settings card pattern)
- Selected icon in icon picker: ring + background highlight (`ring-2 ring-primary bg-primary/10`)
- Selected unit in unit dropdown: active/focus ring (`focus:ring-primary`)
- "Add Section" and "Save Changes" primary CTA buttons: `bg-primary text-primary-foreground`
- Drag-over drop indicator line between cards

**Muted / secondary surface** (`--secondary` / `--muted`): section card expanded area background, meter row hover state (`hover:bg-muted`), archived area background.

**No second accent.** Destructive red is the only second semantic color. It is used exclusively for permanent delete actions (meter delete confirmation, section permanent delete confirmation).

## Interaction States

### Section Card (collapsed)
- **Default:** Card with icon + name (text-sm font-medium) + unit badge (text-xs text-muted-foreground) + drag handle (right side) + edit pencil icon
- **Hover:** `hover:bg-muted/50` on card — subtle highlight
- **Expanded:** chevron-down rotates 180° (CSS transform, 0.2s ease); meter list animates in via Framer Motion `height: 0 → auto`
- **Dragging (active):** `opacity-50` on the dragging card; `boxShadow: '0 8px 24px rgba(0,0,0,0.25)'`; blue drop indicator line (2px, `bg-primary`) appears between items at drop target

### Icon Picker (in dialog)
- **Default cell:** 40×40px rounded-md, `bg-muted`, icon in `text-muted-foreground`
- **Hover:** `hover:bg-accent` — subtle grey highlight
- **Selected:** `ring-2 ring-primary bg-primary/10`, icon in `text-primary`
- Grid: 5 columns (`grid-cols-5`), 8px gap

### Meter Row
- **Default:** text-sm text-foreground, drag handle left, delete icon right (hidden until hover)
- **Hover:** `hover:bg-muted/50` reveals delete icon
- **Inline edit active:** shadcn Input replaces text span, auto-focused, full row width
- **Saving:** no spinner — optimistic; error reverts inline (shows red border on input briefly)

### Unit Dropdown
- **Preset selected:** standard `<select>` styled with Tailwind border/input tokens
- **"Custom..." selected:** text Input appears below dropdown with placeholder "Enter unit"

### Archived Area
- **Collapsed (default):** single row button "Archived (N)" with chevron-right, `text-xs text-muted-foreground`
- **Expanded:** reveals list of archived section rows, each with section name + Restore button (outline) + Delete button (destructive variant, text-xs)

### Delete Confirmation Dialog
- Two instances: meter delete, section permanent delete
- Title: destructive action name (e.g. "Delete Meter")
- Body: warning message copy (see Copywriting Contract)
- Footer: Cancel (outline) + Confirm Delete (destructive variant, `bg-destructive`)

## Animation Contract

| Animation | Implementation | Duration | Easing |
|-----------|----------------|----------|--------|
| Card list stagger entry | Framer Motion `stagger` variants (matches Settings.jsx) | 0.06s stagger, 0.25s per card | `[0.25, 0.1, 0.25, 1]` |
| Section expand/collapse | Framer Motion `AnimatePresence` + `height: 0 → auto` + `opacity: 0 → 1` | 0.2s | `[0.25, 0.1, 0.25, 1]` |
| Archived area expand/collapse | Same as section expand/collapse | 0.2s | `[0.25, 0.1, 0.25, 1]` |
| Chevron rotation | CSS `rotate-180` toggled via className | 0.2s | ease |
| Button tap feedback | `whileTap={{ scale: 0.98 }}` (matches Settings.jsx) | default | spring |
| Drag lift | Instant style (opacity + boxShadow via `isDragging`) | 0ms | none — dnd-kit owns this |
| Drag drop indicator | Instant opacity/height via dnd-kit overlay | 0ms | none |

**Do NOT use Framer Motion `layout` prop on sortable items** (causes double-animation conflict with dnd-kit transforms — see RESEARCH.md Pitfall 3).

**Destructive confirmations summary:**
- Meter delete: one-step — confirmation dialog ("Delete {{name}}? This will permanently remove all entries for this meter.") with Cancel + Delete buttons
