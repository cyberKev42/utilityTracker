**1. Create `frontend/src/components/settings/SectionCard.jsx`:**

Props: `section` (object: { id, name, unit, icon, meters: [] })

Sortable:
- Import `useSortable` from `@dnd-kit/sortable`, `CSS` from `@dnd-kit/utilities`
- `const { attributes, listeners, setNodeRef, transform, transition, isDragging } = useSortable({ id: section.id })`
- Style: `{ transform: CSS.Transform.toString(transform), transition, opacity: isDragging ? 0.5 : 1, boxShadow: isDragging ? '0 8px 24px rgba(0,0,0,0.25)' : undefined }`
- Do NOT use Framer Motion `layout` prop

Collapsed header (default state):
- `flex items-center gap-3 px-4 py-3 min-h-[44px] cursor-pointer` — onClick toggles `isExpanded`
- Left: section icon from `ICON_MAP[section.icon]` rendered at `h-5 w-5 text-primary` (import `ICON_MAP` from `./IconPickerGrid`)
- Center: `<span className="text-sm font-semibold text-foreground flex-1">{section.name}</span>` — render raw, NOT through t(). Then unit badge: `<span className="text-xs text-muted-foreground">{section.unit}</span>`
- Right actions (do NOT spread drag listeners on these):
  - Edit button: `onClick={(e) => { e.stopPropagation(); setEditOpen(true); }}` — render `HiOutlinePencilSquare` at 16px, `aria-label={t('settings.sections.editSectionLabel')}`
  - Archive button: `onClick={(e) => { e.stopPropagation(); archiveSection(section.id); }}` — render `HiOutlineArchiveBox` at 16px, `aria-label={t('settings.sections.archiveSection')}`
  - Drag handle: `<button {...attributes} {...listeners} aria-label={t('settings.sections.reorderHint')}>` — render `HiOutlineBars3` at 16px. Spread listeners ONLY here, not on the card.
- Chevron: `HiOutlineChevronDown` with `className={`transition-transform duration-200 ${isExpanded ? 'rotate-180' : ''}`}`

Expanded area (meters):
- `AnimatePresence` + `motion.div` with `initial={{ height: 0, opacity: 0 }}`, `animate={{ height: 'auto', opacity: 1 }}`, `exit={{ height: 0, opacity: 0 }}`, `transition={{ duration: 0.2, ease: [0.25, 0.1, 0.25, 1] }}`, `style={{ overflow: 'hidden' }}`
- Inside: a SEPARATE `DndContext` + `SortableContext` for meters (scoped, independent from section-level DndContext — per RESEARCH.md anti-pattern warning)
  - `const meterSensors = useSensors(useSensor(PointerSensor), useSensor(KeyboardSensor, { coordinateGetter: sortableKeyboardCoordinates }))`
  - `onDragEnd`: compute `arrayMove(section.meters, oldIndex, newIndex)` and call `reorderMeters(section.id, newOrder)`
  - `SortableContext items={section.meters.map(m => m.id)} strategy={verticalListSortingStrategy}`
  - Render `<MeterRow key={m.id} meter={m} sectionId={section.id} />` for each meter
- "Add Meter" button at bottom: `onClick` calls `createMeter(section.id, { name: 'New Meter' })`. After creation, the new meter appears via context refresh. The MeterRow for the newest meter should ideally start in edit mode — to achieve this, track `newMeterId` state: after `createMeter` resolves, set `newMeterId` to the returned meter id. Pass `isNew={meter.id === newMeterId}` to MeterRow. Clear `newMeterId` after render via useEffect.
- "Add Meter" button: `<Button variant="ghost" size="sm" className="mt-2 text-xs">` with `HiOutlinePlus` icon + `t('settings.sections.addMeter')`

SectionEditDialog integration:
- State: `const [editOpen, setEditOpen] = useState(false)`
- Render: `<SectionEditDialog open={editOpen} onOpenChange={setEditOpen} section={section} />`

Error state for reorder: catch errors from `reorderMeters`, show `text-xs text-destructive` below meter list using `t('settings.sections.reorderError')`, auto-dismiss after 4s.

Wrap entire card in `<Card>` with `ref={setNodeRef}` and the computed style. Add `hover:bg-muted/50` transition.
