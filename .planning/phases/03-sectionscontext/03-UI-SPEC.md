# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: SectionsContext.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase scope note:** Phase 3 is an infrastructure phase. No pages are built.
> The only UI-visible output is two full-page states rendered by SectionsProvider:
> (1) a loading spinner while the initial sections fetch is in-flight, and
> (2) an error banner with Retry when the fetch fails. All other visual work

## Component Inventory

This phase introduces two in-provider UI states only. No new page-level components.

### 1. Full-page Loading Spinner

**When rendered:** While `loading === true` on initial mount (before first sections fetch resolves).

**Implementation:**
```jsx
<div className="min-h-screen flex items-center justify-center">
  <div className="animate-spin rounded-full h-8 w-8 border-2 border-primary border-t-transparent" />
</div>
```

- Spinner: 32px (h-8 w-8), 2px border, primary color with transparent top segment for spin effect
- Centered vertically and horizontally in viewport
- No text label — spinner alone is sufficient for a fast-loading payload

### 2. Full-page Error Banner

**When rendered:** When `error !== null` after fetch failure.

**Implementation:**
```jsx
<div className="min-h-screen flex items-center justify-center">
  <div className="text-center space-y-4 p-8">
    <p className="text-destructive text-base">Failed to load sections. Check your connection and try again.</p>
    <Button onClick={retry} className="min-h-[44px]">Retry</Button>
  </div>
</div>
```

- Uses existing `<Button>` from `src/components/ui/button.jsx`
- Inner padding: p-8 (32px) on all sides
- Vertical gap between text and button: space-y-4 (16px)
- Button minimum height 44px for touch compliance

## Interaction Contract

### Loading State Duration

- Render spinner immediately when `loading === true`
- Remove spinner as soon as `loading === false` (whether fetch succeeded or failed)
- No minimum display duration — do not artificially delay

### Error State Retry

- Clicking Retry calls `fetchSections()` directly
- During retry: Retry button does not need a loading state (the full-page spinner re-renders immediately as `loading` flips to `true`)
- On retry success: spinner replaces error banner, then content renders
- On retry failure: error banner re-renders with same copy

### Optimistic Rollback Toast

- When reorder or rename mutation is rejected by server, throw the error to the calling page
- Calling page is responsible for showing a toast or inline error — Phase 3 does not render a toast itself
- This means Phase 3 has no toast component dependency — toast is Phase 4+ concern
