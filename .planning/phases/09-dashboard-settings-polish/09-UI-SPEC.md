## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (gap-2 within rows) |
| md | 16px | Default element spacing (px-3 within rows = 12px; px-4 for cards) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (enforced via `min-h-[44px]` on MeterRow, `min-height: 44px` on inputs/selects in index.css)
- Toggle control inner buttons: 28px height (h-7) — acceptable because the parent container meets 44px touch target
- Dashboard toggle outer container: `p-1` (4px) padding around inner buttons

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(0 0% 3.5%) — `--background` | Page background |
| Secondary (30%) | hsl(0 0% 6.5%) — `--card` / hsl(0 0% 9%) — `--muted` | Cards, settings rows, toggle container background |
| Accent (10%) | hsl(217 91% 60%) — `--primary` (blue) | Active toggle button background indicator |
| Destructive | hsl(0 72% 51%) — `--destructive` | Delete meter confirmation dialog button only |

Accent (`--primary` blue hsl(217 91% 60%)) reserved for:
- Active state of the usage/cost toggle button (the selected pill background — matches how LanguageSwitcher uses `bg-accent text-foreground` for active lang)

Note: The LanguageSwitcher active state uses `bg-accent` (hsl(0 0% 11%)) not primary blue. The Dashboard toggle should match this exact pattern — `bg-background shadow-sm` on active pill over `bg-muted` container — as specified in RESEARCH.md Pattern 1 code example. Primary blue (`--primary`) is reserved for ring/focus states and selection highlight only.

Toggle active state — precise values:
- Container: `bg-muted` (hsl(0 0% 9%)) with `p-1 rounded-lg`
- Active pill: `bg-background text-foreground shadow-sm` (hsl(0 0% 3.5%))
- Inactive pill: `text-muted-foreground hover:text-foreground`

Pencil icon affordance (MeterRow SETT-01):
- Default: `opacity-0` (hidden, same as existing mode/delete buttons)
- On `group-hover`: `opacity-100 text-muted-foreground`
- On button hover: `text-foreground`
- Pattern matches existing mode-toggle and delete buttons in MeterRow lines 126-143

Water icons in IconPickerGrid (VISL-01):
- Rendered at same size as all other icons in the picker grid
- No special color — inherits `text-foreground` / `text-muted-foreground` from grid item state
- Selected state: existing grid item selected style (unchanged)

Source: `index.css` CSS variable definitions, `LanguageSwitcher.jsx`, `MeterRow.jsx`.

| Water icon aria-labels | Icon component names used as aria-labels via existing IconPickerGrid pattern — no new copy |

Dashboard toggle placement: above section cards, left-aligned, same row as the page heading or directly below it. Label "Cost" / "Usage" — no additional explanatory text. The toggle is self-explanatory at this scope.

Usage value format: `{Number(total_usage || 0).toLocaleString()} {unit}` — example output: "42.3 m³". No label prefix needed; the toggle label already communicates the mode.

"Total Spending" summary card: always displays cost. No copy change. Toggle does not affect this card (see RESEARCH.md Pitfall 1).

## Interaction Contracts

### DASH-01: Usage/Cost Toggle

- Toggle is a segmented two-button control (pill style), not a checkbox or switch
- Both states are always visible; active state is visually distinct (elevated background)
- Preference persists via `localStorage` key `dashboard_display_mode` (default: `'cost'`)
- Toggle state switches immediately on click — no loading state, no animation delay beyond `transition-colors duration-150`
- `motion.button` with `whileTap={{ scale: 0.98 }}` on each pill button
- Section cards animate value change via existing card render cycle — no additional animation required
- Toggle does NOT affect the "Total Spending" summary card

### SETT-01: Meter Inline Edit Affordance

- Pencil icon (`HiOutlinePencilSquare`, 16px / h-4 w-4) appears on row hover via `opacity-0 group-hover:opacity-100 transition-opacity`
- Pencil button positioned between mode-toggle button and delete button in the right-side action group
- Clicking pencil sets `editing=true` — same effect as clicking the meter name span
- `e.stopPropagation()` on pencil button click to prevent drag initiation
- Existing meter name span retains `onClick={() => setEditing(true)}` — pencil is additive, not a replacement
- Edit field behavior unchanged: `onBlur` saves, `Enter` saves, empty value reverts
- No animation on edit activation — Input replaces span directly

### VISL-01: Water Icon Expansion

- 8 new icons added to `ICON_MAP`: `FaDroplet`, `FaWater`, `FaFaucet`, `FaFaucetDrip` (fa6), `LuDroplet`, `LuDroplets`, `LuGlassWater`, `LuWaves` (lu)
- Icons rendered at same size as existing icons in picker grid
- No new grouping or filtering UI — icons appear in the existing flat grid
- Existing guard in `SectionCard.jsx` (`ICON_MAP[section.icon] || null`) remains unchanged — backward-compatible
