Create `frontend/src/context/EntriesDataContext.jsx`:
- `createContext(null)` exported as `EntriesDataContext`
- `EntriesDataProvider({ children })` component:
  - Get `user` from `useAuth()`
  - State: `stats` (null), `entries` ([]), `trend` (null), `loading` (true — only for first load), `error` (null)
  - `fetchAll` useCallback: calls `Promise.all([getStats(), getEntries({ limit: 500 }), getTrend()])`, sets all three state values, sets `loading = false` in finally block. Does NOT set `loading = true` (only the initial useState(true) triggers the loading state — per D-01, D-02)
  - `revalidate` useCallback: calls `fetchAll()` without setting loading=true — background refresh, no spinner
  - useEffect: if `!user`, reset state and return. Otherwise call `fetchAll()`. Deps: `[user, fetchAll]`
  - `recentEntries` derived via `useMemo`: `entries.slice(0, 5)` — so Dashboard gets recent entries without re-slicing
  - Value object via useMemo: `{ stats, entries, recentEntries, trend, loading, error, revalidate }`
  - Render: `<EntriesDataContext.Provider value={value}>{children}</EntriesDataContext.Provider>` — NO loading guard/spinner at provider level (children always render)

Create `frontend/src/hooks/useEntriesData.js`:
- Import `useContext` from react, import `EntriesDataContext` from context
- Export `useEntriesData()`: returns `useContext(EntriesDataContext)`, throws Error if null ("useEntriesData must be used within EntriesDataProvider")

Wire into App.jsx:
- Import `EntriesDataProvider` from `../context/EntriesDataContext`
- Wrap inside the existing provider tree, after `SectionsProvider` (needs auth context): `<SectionsProvider><EntriesDataProvider>...existing children...</EntriesDataProvider></SectionsProvider>`
