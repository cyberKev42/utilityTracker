**Helper function (internal, not exported):**

`splitIntoDailyRows(startDate, endDate, usageAmount, unitPrice)` — Computes day count as `Math.round((end - start) / 86400000) + 1` (inclusive). Per-day usage = `Math.round((usageAmount / days) * 100) / 100`. Per-day cost = `Math.round((usageAmount * unitPrice / days) * 100) / 100`. Last row absorbs remainder for both usage and cost. Returns array of `{ date: 'YYYY-MM-DD', usage_amount, cost_amount }`.

**Exported functions:**

1. `createEntry(userId, { meter_id, start_date, end_date, usage_amount, meter_reading, unit_price })`:
   - Ownership check: `SELECT m.id, m.entry_mode FROM utility_meters m JOIN utility_sections s ON s.id = m.section_id WHERE m.id = $1 AND s.user_id = $2`. Return 404-status error if no rows.
   - Get unit_price: If `unit_price` not provided, look up from utility_settings: `SELECT unit_price FROM utility_settings WHERE user_id = $1 AND section_id = (SELECT section_id FROM utility_meters WHERE id = $2)`. Default to 0 if not found.
   - **If entry_mode = 'meter_reading':**
     - Require `meter_reading` field (throw 400 if missing).
     - Query prior reading: `SELECT meter_reading, date FROM utility_entries WHERE meter_id = $1 ORDER BY date DESC, created_at DESC LIMIT 1`.
     - `is_initial = prior.rows.length === 0`.
     - `usage_amount = is_initial ? 0 : Math.round((meter_reading - prior.rows[0].meter_reading) * 100) / 100`.
     - `cost_amount = Math.round(usage_amount * effectiveUnitPrice * 100) / 100`.
     - INSERT single row with meter_reading, usage_amount, cost_amount, date=start_date, user_id. RETURNING *.
     - Add `is_initial` to returned object.
   - **If entry_mode = 'usage_amount':**
     - Require `usage_amount` field (throw 400 if missing).
     - If `start_date === end_date` (single day): INSERT one row. `cost_amount = Math.round(usage_amount * effectiveUnitPrice * 100) / 100`. RETURNING *.
     - If `start_date !== end_date` (range): Generate `group_id = crypto.randomUUID()`. Call `splitIntoDailyRows(start_date, end_date, usage_amount, effectiveUnitPrice)`. Use transaction: INSERT each row with shared group_id, meter_id, user_id. Return array of created entries.
   - Add `import { randomUUID } from 'node:crypto';` at top of file.

2. `deleteEntry(userId, entryId)`:
   - Fetch the entry: `SELECT e.*, m.entry_mode FROM utility_entries e JOIN utility_meters m ON m.id = e.meter_id WHERE e.id = $1 AND e.user_id = $2`. Return 404 if not found.
   - **If entry_mode = 'meter_reading':** Use transaction. Find next entry: `SELECT id, meter_reading FROM utility_entries WHERE meter_id = $1 AND (date > $2 OR (date = $2 AND created_at > $3)) ORDER BY date ASC, created_at ASC LIMIT 1`. If next exists, find entry before deleted: `SELECT meter_reading FROM utility_entries WHERE meter_id = $1 AND (date < $2 OR (date = $2 AND created_at < $3)) ORDER BY date DESC, created_at DESC LIMIT 1`. Compute new delta: if no before entry, newDelta=0; else newDelta = next.meter_reading - before.meter_reading. UPDATE next entry's usage_amount = `Math.round(newDelta * 100) / 100`. Also update cost_amount by looking up unit_price from settings. Then DELETE the entry.
   - **If entry_mode = 'usage_amount':** Simple DELETE WHERE id=$1 AND user_id=$2.
   - Return `{ deleted: true }`.

3. `getEntries(userId, { meter_id, section_id, year, month, limit, offset })`:
   - Build query dynamically. Base: `SELECT e.*, m.name as meter_name, s.name as section_name, s.unit FROM utility_entries e JOIN utility_meters m ON m.id = e.meter_id JOIN utility_sections s ON s.id = m.section_id WHERE e.user_id = $1`.
   - Add optional filters: `AND e.meter_id = $N`, `AND m.section_id = $N`, `AND EXTRACT(YEAR FROM e.date) = $N`, `AND EXTRACT(MONTH FROM e.date) = $N`.
   - ORDER BY e.date DESC, e.created_at DESC. LIMIT/OFFSET if provided.
   - Return rows.

4. `getStats(userId, { year, month })`:
   - Two queries merged in JS (pattern from RESEARCH.md):
   - Section totals: `SELECT s.id, s.name, s.unit, COALESCE(SUM(e.usage_amount), 0)::float AS total_usage, COALESCE(SUM(e.cost_amount), 0)::float AS total_cost, COUNT(e.id)::int AS entry_count FROM utility_sections s LEFT JOIN utility_meters m ON m.section_id = s.id AND m.archived_at IS NULL LEFT JOIN utility_entries e ON e.meter_id = m.id {dateFilter} WHERE s.user_id = $1 AND s.archived_at IS NULL GROUP BY s.id, s.name, s.unit ORDER BY s.sort_order`.
   - Meter totals: `SELECT m.id, m.name, m.section_id, COALESCE(SUM(e.usage_amount), 0)::float AS total_usage, COALESCE(SUM(e.cost_amount), 0)::float AS total_cost, COUNT(e.id)::int AS entry_count FROM utility_meters m LEFT JOIN utility_entries e ON e.meter_id = m.id {dateFilter} JOIN utility_sections s ON s.id = m.section_id WHERE s.user_id = $1 AND m.archived_at IS NULL AND s.archived_at IS NULL GROUP BY m.id, m.name, m.section_id`.
   - Date filter: If year provided, add `AND EXTRACT(YEAR FROM e.date) = $N`. If month also provided, add `AND EXTRACT(MONTH FROM e.date) = $N`.
   - Merge: build `metersBySection` map, return `sectionStats.rows.map(s => ({ ...s, meters: metersBySection[s.id] ?? [] }))`.
   - Wrap in `{ sections: [...] }`.
