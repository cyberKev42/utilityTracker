**SectionsProvider component** — exported named function that:

1. Calls `useAuth()` to get `user`, calls `useTranslation()` to get `t`
2. State: `sections` (array, default []), `loading` (boolean, default true), `error` (null or Error)
3. `fetchSections` wrapped in useCallback (empty deps) — sets loading true, error null, tries `sectionsService.getSections()`, sets sections on success, sets error on failure, sets loading false in finally
4. useEffect watching `[user, fetchSections]`:
   - If `!user`: `setSections([])`, `setError(null)`, `setLoading(false)`, return early
   - Otherwise: call `fetchSections()`
5. **Server-wait mutations** (create, update, delete, archive, unarchive, createMeter, updateMeter, deleteMeter) — each wrapped in useCallback, calls the corresponding sectionsService function, then awaits `fetchSections()`, returns the result. Dependencies: `[fetchSections]`.
6. **Optimistic mutations:**
   - `reorderSections(newOrder)` — snapshot `const previous = sections`, optimistically `setSections(newOrder)`, try `await sectionsService.reorderSections(newOrder.map(s => s.id))`, catch: `setSections(previous)` then `throw err`. Deps: `[sections]`.
   - `updateSection` should be server-wait (not optimistic) per decision — rename is "cosmetic" but updateSection also changes unit/icon so keep it server-wait for simplicity.
   - `reorderMeters(sectionId, newMeters)` — snapshot sections, optimistically update the section's meters array in state, try `await sectionsService.reorderMeters(sectionId, newMeters.map(m => m.id))`, catch: restore snapshot and throw. Deps: `[sections]`.
7. **Helper lookups** — each wrapped in useCallback with `[sections]` dep:
   - `getSectionById(id)` — `sections.find(s => s.id === id) ?? null`
   - `getMeterById(id)` — iterate sections, find meter by id, return or null
   - `getSectionForMeter(meterId)` — `sections.find(s => s.meters?.some(m => m.id === meterId)) ?? null`
8. `fetchWithArchived` — useCallback with empty deps, returns `sectionsService.getSectionsWithArchived()` directly (does NOT update context state)
9. **Context value** — useMemo wrapping object with: `sections`, `loading`, `error`, `retry: fetchSections`, all mutation functions, all helper lookups, `fetchWithArchived`. Dependencies: all the values and functions.
10. **Render logic** (inside the return):
    - If `loading` is true: render full-page spinner per UI-SPEC:
      ```jsx
      <div className="min-h-screen flex items-center justify-center">
        <div className="animate-spin rounded-full h-8 w-8 border-2 border-primary border-t-transparent" />
      </div>
      ```
    - If `error` is not null: render error banner per UI-SPEC:
      ```jsx
      <div className="min-h-screen flex items-center justify-center">
        <div className="text-center space-y-4 p-8">
          <p className="text-destructive text-base">{t('sections.loadError', 'Failed to load sections.')}</p>
          <p className="text-muted-foreground text-sm">{t('sections.loadErrorHint', 'Check your connection and try again.')}</p>
          <Button onClick={fetchSections} className="min-h-[44px]">{t('sections.retry', 'Retry')}</Button>
        </div>
      </div>
      ```
    - Otherwise: render `<SectionsContext.Provider value={value}>{children}</SectionsContext.Provider>`
