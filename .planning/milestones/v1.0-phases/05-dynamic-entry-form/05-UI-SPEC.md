## Spacing Scale

Declared values (must be multiples of 4). All values sourced from existing AddEntry.jsx patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap within inline elements (gap-1) |
| sm | 8px | Space between label and field error message (space-y-2) |
| md | 16px | Default form field stack gap (space-y-4) |
| lg | 20px | Form section gap — overrides standard 24px for compact form density (space-y-5, matches existing AddEntry.jsx) |
| xl | 24px | Card internal padding top/bottom, section heading margin (p-6, mb-8 is 32px) |
| 2xl | 32px | Page heading bottom margin (mb-8 = 32px) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for all `<input>`, `<select>`, `<button>` elements: minimum 44px height (enforced globally in `index.css` via `min-height: 44px` on `input[type="date"]`, `input[type="number"]`, `select`)
- Card content padding: `p-5` (20px) on mobile, `sm:p-6` (24px) on sm+ breakpoint — matches existing AddEntry.jsx
- **20px (lg token) is a codebase-inherited exception:** a multiple of 4 used consistently in the existing AddEntry.jsx form stack; retained to match established density rather than substituting the standard 24px step.

Source: `frontend/src/pages/AddEntry.jsx` class patterns, `frontend/src/index.css`

## Typography

All values extracted from existing AddEntry.jsx and index.css. Do not introduce new sizes or weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 14px (text-sm) | 600 (semibold — override `label.jsx` to use `font-semibold`) | 1.4 |
| Heading | 20px (text-xl) | 600 (semibold — font-semibold) | 1.25 |
| Caption / meta | 13px (text-[13px]) | 400 (regular) | 1.5 |

Notes:
- Two weights in use: 400 (regular) and 600 (semibold). The shadcn Label component defaults to `font-medium` (500) — override it with `font-semibold` (className prop or component-level default) so only 400 and 600 are rendered. Do not rely on the component default.
- 13px is reserved for page subtitle/description lines only (e.g. page sub-heading below h1). Do not use for field labels or errors.
- Error messages: 12px (text-xs) at weight 400, color `text-destructive` — matches existing pattern in AddEntry.jsx.

Source: `frontend/src/pages/AddEntry.jsx` (lines 243–244), `frontend/src/index.css`

| Role | Token | Resolved (approx.) | Usage |

| Muted surface | `hsl(var(--muted))` | hsl(0 0% 9%) | Info banner background, split preview callout background (bg-muted) |

Accent reserved for:
1. Primary submit button (`Button` default variant — `bg-primary`)

4. Split preview callout — use `bg-accent/40 border border-border/30` (existing pattern from RESEARCH.md code example, not primary blue)

Do NOT use primary blue on: section dropdown labels, meter dropdown, date fields, unit labels, info banner, last-reading display text.

## Interaction Contract

### Cascading Section → Meter Dropdowns

- Section dropdown: always visible, full width, placeholder "Select a section"
- Meter dropdown: only rendered after a section is selected (`sectionId !== ''`)
- On section change: reset meterId, usage/reading fields, fieldErrors, touched — meter dropdown disappears then reappears with new meter list
- Transition: no animation on dropdown appearance (native select, instant show/hide via conditional render)
- Both dropdowns use the same `select` class as the existing meter selector in AddEntry.jsx

### Entry Mode Switching

- Mode is determined automatically by `selectedMeter.entry_mode` — no visible toggle
- When mode changes (meter selection changes), the mode-specific field block animates in via `AnimatePresence` + `motion.div` with `initial={{ opacity: 0, height: 0 }} animate={{ opacity: 1, height: 'auto' }} exit={{ opacity: 0, height: 0 }}`
- Usage mode shows: start date, end date, usage amount field
- Reading mode shows: single date field (start date only), meter reading field

### Date Range Split Preview

- Appears below the date fields in usage mode only
- Condition: startDate and endDate are both valid, endDate >= startDate, startDate !== endDate, usageAmount > 0
- Container: `rounded-lg bg-accent/40 border border-border/30 px-4 py-3`
- Text: `text-sm text-muted-foreground`
- Animates in/out with `AnimatePresence` + `motion.div` (same pattern as existing cost preview)
- No separate label — inline sentence format only

### Last Reading Display

- Appears below the meter dropdown when a reading-mode meter is selected and last-reading API returns a non-null result
- Format: small caption line `text-[13px] text-muted-foreground` below the meter select
- Content: "Last reading: {value} {unit} on {date}"
- No card or border — plain text

### First-Reading Info Banner

- Appears when `isReadingMode && lastReadingLoaded && lastReading === null`
- Container: `flex items-start gap-2 p-3 rounded-lg bg-muted border border-border/50`
- Icon: `HiInformationCircle` at `h-5 w-5 text-muted-foreground shrink-0 mt-0.5`
- Text: `text-sm text-muted-foreground`
- No close/dismiss button — disappears when meter is deselected or a reading is submitted

### Cost Preview

- Appears below the usage amount field when `unit_price` is available (from section data if exposed, otherwise from optional manual input field)
- Container: existing pattern — `text-sm text-muted-foreground` inline below field
- Format: "Estimated cost: {formatCurrency(cost)}"
- Animates in/out with same `AnimatePresence` pattern as existing AddEntry.jsx

### Server Error Banner

- Full-width banner above the form fields inside the card
- Container: `flex items-start gap-2 p-3 mb-5 rounded-lg bg-destructive/10 border border-destructive/20`
- Icon: `HiExclamationCircle` at `h-5 w-5 text-destructive shrink-0 mt-0.5`
- Text: `text-sm text-destructive`
- Animates in with `AnimatePresence` + `motion.div`, `initial={{ opacity: 0, height: 0 }}`

### Field Validation Errors

- Inline below each field: `text-xs text-destructive mt-1`
- Only shown when `touched[field] && fieldErrors[field]`
- Error border on field: `border-destructive focus-visible:ring-destructive`
- Triggered on blur per field, and on all fields at once on submit attempt

### Success State

- Replaces the entire page content (not modal overlay)
- Card with `p-6` content, centered column layout
- Spring-animated checkmark icon: `h-14 w-14 rounded-xl bg-primary/10` container, `h-8 w-8 text-primary` icon
- Success text: `text-lg font-semibold text-foreground`
- "Add Another" button: `Button` default variant, `size="lg"`
- "Add another" resets ALL fields including section and meter selection

### Loading State

- Submit button shows disabled state while loading (`disabled` attribute)
- No spinner required — button disables is sufficient

## Layout

- Page max-width: `max-w-lg mx-auto` (512px centered)
- Page heading block: `mb-8` (32px margin-bottom)
  - h1: `text-xl font-semibold text-foreground tracking-tight`
  - Subtitle: `text-[13px] text-muted-foreground mt-1`
- Form fields: single column, `space-y-5` stack
- Card: single card wrapping all form content
- Responsive: no layout change between mobile/desktop — single column always. Card padding adjusts from `p-5` (mobile) to `sm:p-6` (640px+)
- **Primary visual anchor:** The "Save Entry" submit button is the terminal focal point of the single-column layout. It appears last in the card, full-width, at `Button` default (primary blue) variant — the only accent-colored element in the form body.

| Last usage display | `addEntry.lastUsage` | "Last entry: {{value}} {{unit}} on {{date}}" |
| First-reading banner | `addEntry.firstReadingInfo` | "This is the first reading for this meter. Enter the current value — usage will start from the next entry." |
| Split preview | `addEntry.splitPreview` | "{{total}} {{unit}} over {{days}} days = {{perDay}} {{unit}}/day" |
| Primary CTA | `addEntry.submit` | already exists — verify it reads "Save Entry" |
| Success message | `addEntry.success` | already exists — no change |
| Add another button | `addEntry.addAnother` | already exists — no change |
| No meters empty state | `addEntry.noMeters` | already exists — no change |

**Primary CTA:** "Save Entry" (existing key `addEntry.submit` — confirm wording during implementation)

**Empty state (no sections/meters):** Existing `addEntry.noMeters` key — centered `text-sm text-muted-foreground`, no icon, no action button. No change needed.

**Empty state (section selected, no meters):** "No meters in this section. Add one in Settings." — key `addEntry.noMetersInSection`. Links or navigates to settings not required — plain informational text only.

**Error state — field validation:**
- Meter not selected: "Please select a meter" (existing key `addEntry.validation.meterRequired`)
- Date missing: "Date is required" (existing key `addEntry.validation.dateRequired`)
- End date before start: "End date must be after start date" (existing key `addEntry.validation.endDateBeforeStart`)
- Usage amount missing or zero: "Enter a usage amount greater than 0" (existing key `addEntry.validation.usageRequired` / `usagePositive`)
- Meter reading negative: "Reading must be 0 or greater" (existing key `addEntry.validation.meterReadingNonNegative`)

**Error state — server error:** Display raw `err.message` from API response inside destructive banner. No generic fallback needed — backend returns descriptive errors.

**Destructive actions in this phase:** None. Entry submission is not destructive. No confirmation dialogs required.
